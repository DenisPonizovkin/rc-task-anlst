# Данные

В файле data.csv хранится информация о числе объектов,
подлежащих обновлению и стоимостях инвестиционных мероприятиях
по обвнолению объектов.

Существует конечное число типов объектов и мероприятий, у каждого из которых есть свой
целочисленный идентификатор. В приведенных данных тип объекта соответствует полю
```object_tp_id```, тип мероприятия - полю ```arrangement_tp_id```.

Информация по числу объектов и величине инвестиций по их обновлению  обладает
однотипной структурой, поэтому хранится в единой таблице, но различается
по типу, зафиксированному в поле  ```te_form_tp_id```. Значение этого поля равное:
1324 - определеяет величину, соответствующую числу объектов,
подлежащих обновлению (единицы измерения - штуки)
1326 - определяет величину, соответствующую затратам по обновлению объектов

Эти два среза данных пересекаются по полям:
- ```object_tp_id``` - тип объекта обновления
- ```arrangement_tp_id``` - тип мероприятия обновления объекта

Приведенные данные являются изначальной справочной информацией.
При этом значения колонки val для 
```te_form_tp_id``` = 1326 являются неизменной величиной. То есть число объектов является
переменной варьируеморй величиной при решении задачи. 

Обозначим как А срез данных, соответствующий ```te_form_tp_id``` = 1324.
Обозначим как Б срез данных, соответствующий ```te_form_tp_id``` = 1326.
А х Б - пересечение (inner join) по полям ```object_tp_id``` и ```arrangement_tp_id```.

# Задача

Введем функцию И(А, Б)=сумма(А.val*Б.val), А,Б \in А х Б - вещественное число,
равное сумме произведений инвестиционных затрат на соответствующее число объектов
по типу объектов и мероприятий (пересечение (inner join)
по полям ```object_tp_id``` и ```arrangement_tp_id```).

И(А,Б) от изначальных А,Б является максимумом.

Необходимо уменьшить количество объектов различныъх типов ```object_tp_id``` так, чтобы
это количество объектов стремилось к максимуму, а разница
( И(А,Б) - И(А,Б_1) ) стремилась к минимуму, где Б_1 - данные, соответствующие
уменьшенненному числу объектов.
